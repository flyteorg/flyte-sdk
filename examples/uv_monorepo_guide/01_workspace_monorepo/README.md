# 01_shared_lockfile_monorepo

A monorepo with two library packages (`lib_transforms`, `lib_models`) and a main application (`workspace_app`), all under a single `src/` directory. Demonstrates using dependency groups to produce separate, lean images for ETL and ML workloads from a single shared lockfile.

## What this example shows

- One `uv.lock` at the project root covers all packages
- Two `TaskEnvironment` objects, each installing only its external PyPI deps via `--only-group`
- `etl_env` installs `pandas` only; `ml_env` installs `scikit-learn` only
- `lib_transforms` and `lib_models` are not listed as dependencies — their source travels via the code bundle
- `extra_args="--only-group etl"` is part of the image hash, so ETL and ML produce distinct images despite sharing the same `pyproject.toml`
- `root_dir = src/` so the code bundle covers all packages under `src/`

## Setup

```bash
cd 01_workspace_monorepo
uv sync
```

## Run (fast deploy — development)

```bash
uv run python src/workspace_app/main.py
```

## Run (full build — production)

Uncomment the `flyte.deploy(...)` lines in `main.py` (and set a version), then:

```bash
uv run python src/workspace_app/main.py
```

## Why two TaskEnvironments?

Without `--only-group`, a single environment would install all dependencies into one image. Splitting by group keeps each image small. The ETL image does not include scikit-learn; the ML image does not include pandas.

## Project structure

```
01_shared_lockfile_monorepo/
+-- pyproject.toml                <- defines dependency groups
+-- uv.lock                       <- generated by `uv lock`
+-- src/
    +-- workspace_app/
    |   +-- __init__.py
    |   +-- main.py               <- entry point
    |   +-- tasks/
    |       +-- __init__.py
    |       +-- envs.py           <- TaskEnvironment definitions
    |       +-- etl_tasks.py      <- ETL tasks
    |       +-- ml_tasks.py       <- ML tasks
    +-- lib_transforms/
    |   +-- __init__.py
    |   +-- ops.py                <- normalize, moving_average
    +-- lib_models/
        +-- __init__.py
        +-- baseline.py           <- mean predictor
```
